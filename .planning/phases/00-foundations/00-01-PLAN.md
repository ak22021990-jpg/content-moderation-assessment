      - On match (grep exit 0): echo rejection message and exit 1

      - actions/checkout@v6 with fetch-depth: 0
      - "Scan for forbidden brand strings" step that runs grep with flags:
          -riE "$FORBIDDEN"
          --include="*.js" --include="*.jsx" --include="*.ts" --include="*.tsx"
          --include="*.html" --include="*.json" --include="*.md" --include="*.yml"
          --include="*.yaml" --include="*.css"
          --exclude-dir=node_modules --exclude-dir=.git
          --exclude=package-lock.json
          . (dot = search entire repo)
        Then chain: && echo "BRAND LEAK DETECTED" && exit 1 || echo "Brand check passed"
        (grep exits 0 on FIND, 1 on not-found — the && exit 1 || echo chain means: found=fail, not-found=pass)
    The FORBIDDEN variable in the CI step must match the pattern in docs/brand-guardrails.md exactly.
