  <action>
    Work in C:\Users\anoop\projects\content-moderation-assessment.

    Create .github/workflows/deploy.yml with this exact structure:

    name: Deploy
    (workflow-level permissions block — BOTH pages: write AND id-token: write are required. The deploy job requests an OIDC token; without id-token: write at workflow level the job cannot get one, even if the job-level environment is set correctly.)
    permissions:
      contents: read
      pages: write
      id-token: write
    concurrency:
      group: pages
      cancel-in-progress: true
    on:
      push:
        branches: [main]
      workflow_dispatch:

    jobs:
      build:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v6
            with:
              lfs: true          (DEPLOY-04: required to resolve LFS pointer files to real content)
          - uses: actions/setup-node@v4
            with:
              node-version: 20
              cache: npm
          - run: npm ci
          - run: npm run build
          - name: Verify LFS files resolved
            run: |
              file public/videos/placeholder.mp4 | grep -i "ISO Media\|mp4"
              if [ $? -ne 0 ]; then
                echo "ERROR: LFS pointer file detected instead of real MP4."
                echo "Ensure lfs: true is set in actions/checkout step."
                exit 1
              fi
          - uses: actions/configure-pages@v5
          - uses: actions/upload-pages-artifact@v3
            with:
              path: ./dist

      deploy:
        needs: build
        runs-on: ubuntu-latest
        environment:
          name: github-pages
          url: ${{ steps.deployment.outputs.page_url }}
        steps:
          - id: deployment
            uses: actions/deploy-pages@v4

    Note: The "Verify LFS files resolved" step will fail on this first push because placeholder.mp4 does not yet exist (it is committed in Plan 03). This is expected — the step validates LFS retrieval correctly once Plan 03 pushes the video. The step must be present in the workflow from this commit forward.

    After creating the file, before committing, enable GitHub Pages:
      Open https://github.com/ak22021990-jpg/content-moderation-assessment/settings/pages
      Under "Build and deployment" → Source: select "GitHub Actions" (NOT "Deploy from a branch").
      Save the setting.

    Then commit and push:
      git add .github/workflows/deploy.yml
      git commit -m "feat: github actions deploy pipeline"

    The brand-guard pre-commit hook will run. "Deploy" as a workflow name is generic. Verify the yml file contains no forbidden strings.

    After push: monitor https://github.com/ak22021990-jpg/content-moderation-assessment/actions

  </action>
