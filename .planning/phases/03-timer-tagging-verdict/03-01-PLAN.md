      elapsed = performance.now() - startedAt
      remainingMs = Math.max(0, 180000 - elapsed)

    isComplete: false

<risks>
Wave-specific risks:

1. **RAF mock behaviour differs from browser.** happy-dom does not run RAF natively — mock must accurately simulate synchronous callback execution. If mock doesn't replicate `requestAnimationFrame(laterRafId = requestAnimationFrame(fn))` loop pattern, tick tests fail. Mitigation: RESEARCH.md §10 mock pattern tested in flagmail1 reference project; identical approach.
2. **sessionStorage unavailable in test environment.** happy-dom provides sessionStorage but some setups clear it between tests. Mitigation: beforeEach resets store state via setState() directly; sessionStorage tests explicitly call safeWriteTimer → safeReadTimer.
3. **performance.now() mock timing.** If vi.spyOn returns static value, elapsed time doesn't accumulate across ticks. Mitigation: tick tests mock now to return increasing values matching expected elapsed.
