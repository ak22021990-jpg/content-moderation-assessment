tagReducer.js — pure function (RESEARCH.md §5):
  initialState: { selectedL1: [], selectedL2: [] }

  TOGGLE_L1 action (id: string):
    if selectedL1 includes id: remove id, then drop all L2s where l2Id.split('.')[0] === id
    else: add id to selectedL1
    Return: { ...state, selectedL1: next, selectedL2: keptL2 }

  TOGGLE_L2 action (id: string):
    if selectedL2 includes id: remove, else: add
    Return: { ...state, selectedL2: next }

  RESET action:
    Return: initialState

      ```
      export const initialTagState = { selectedL1: [], selectedL2: [] }

      export function tagReducer(state, action) {
        switch (action.type) {
          case 'TOGGLE_L1': {
            const l1Id = action.id
            if (state.selectedL1.includes(l1Id)) {
              const nextL1 = state.selectedL1.filter(id => id !== l1Id)
              const keptL2 = state.selectedL2.filter(l2Id => {
                const parentL1 = l2Id.split('.')[0]
                return nextL1.includes(parentL1)
              })
              return { ...state, selectedL1: nextL1, selectedL2: keptL2 }
            }
            return { ...state, selectedL1: [...state.selectedL1, l1Id] }
          }
          case 'TOGGLE_L2': {
            const l2Id = action.id
            if (state.selectedL2.includes(l2Id)) {
              return { ...state, selectedL2: state.selectedL2.filter(id => id !== l2Id) }
            }
            return { ...state, selectedL2: [...state.selectedL2, l2Id] }
          }
          case 'RESET':
            return initialTagState
          default:
            return state
        }
      }
      ```

<risks>
Wave-specific risks:

1. **L2 ID parsing assumption.** tagReducer assumes L2 IDs are "X.Y" — extracts parent via `split('.')[0]`. If taxonomy.json uses different ID format (e.g., "1-1"), cascade logic breaks. Mitigation: verified — taxonomy.json uses "X.Y" format throughout (e.g., "1.1", "1.2").
2. **10 L1 categories overflow viewport.** On smaller screens, L1 list may be too tall without scroll. Mitigation: L1 list has max-height: 65vh with overflow-y: auto. Desktop-first design (960px max-width). If responsive breaks, CSS-only fix — no JS changes needed.
3. **Keyboard navigation between L1 chips.** Tab goes L1→L2→next L1 in DOM order. This is acceptable (fieldset grouping reads well in screen-readers). If user feedback demands arrow-key navigation between L1s, it's a Phase 6 polish item.
4. **aria-describedby ID collisions.** Each L2's hidden description span has id="l2-{id}-desc". IDs like "1.1" contain dots which are valid in HTML5 `id` attributes but may confuse CSS selectors. Mitigation: use `id={l2-${sub.id.replace(/\./g, '-')}-desc}` or keep dot — HTML5 allows dots. Safer to replace dots with hyphens: `id={l2-1-1-desc}`.
5. **Taxonomy.json version mismatch.** If taxonomy.json is updated with new L1/L2 IDs after tests are written, tests hardcoding specific IDs break. Mitigation: tests use taxonomy.json import to generate test data (`const firstL1Id = taxonomy.categories[0].id`) rather than hardcoding "1".
</risks>
