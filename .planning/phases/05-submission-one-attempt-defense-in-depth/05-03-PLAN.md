    When `BACKEND === 'formspree'`:
    - Use endpoint: `https://formspree.io/f/${FORMSPREE_FORM_ID}`
    - Payload is the same buildSubmissionPayload object but WITHOUT the `hmac` field (Formspree does not validate HMAC)
    - Submit via raw `fetch` to the Formspree endpoint (do NOT use `@formspree/react`'s `useForm` hook inside `submitResults` — that hook is for React form components. `submitResults` is a pure async function that uses raw fetch.)

    - On success (200): Formspree returns `{ ok: true }` or `{ next: "..." }` — return `{ ok: true, id: 'formspree-' + Date.now() }` (Formspree has no rowId concept)
    - On failure: same retry/error behavior as Apps Script path
    - Important: Formspree mode has NO HMAC signing, NO dedup, NO Sheet writing. Document in code comment: "Formspree fallback — degraded path. No server-side HMAC validation, no dedup, no Google Sheet row. For temporary quota overflow only."

    When `BACKEND === 'apps-script'` (default):
    - Existing behavior from Plan 05-02 is unchanged

    The branching logic:
    ```
    const BACKEND = (typeof import.meta !== 'undefined' && import.meta.env?.VITE_SUBMISSION_BACKEND) || 'apps-script';
    const endpoint = BACKEND === 'formspree'
      ? `https://formspree.io/f/${FORMSPREE_FORM_ID}`
      : APPS_SCRIPT_URL;

    // If using Formspree, strip HMAC field from payload
    const body = BACKEND === 'formspree'
      ? (() => { const { hmac, ...rest } = payload; return rest; })()
      : payload;
    ```
