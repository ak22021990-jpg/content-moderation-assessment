## Component Inventory

### New Components for Phase 5

#### 1. `SubmissionOverlay` (new: `src/components/scoreboard/SubmissionOverlay.jsx`)

States: four visually distinct phases.

| Phase | Visual | Interaction |
|-------|--------|-------------|
| `idle` | Not rendered. Scoreboard visible normally. | N/A |
| `submitting` | Full-screen overlay: `rgba(10,11,20,0.85)` backdrop over scoreboard. Centered card (max-width 400px, padded `--cma-space-lg`) containing: a CSS spinner (20px × 20px, 2px solid `--cma-spinner-color` border, infinite spin animation), heading "Submitting your results", body text, attempt counter text in `--cma-text-secondary`. | Scoreboard content visible underneath but obscured. User cannot interact with scoreboard. No dismiss action. |
| `success` | Same overlay backdrop. Card background tinted `--cma-success` at 10% opacity (CSS: `background: rgba(16,185,129,0.1)` with `--cma-bg-elevated` fallback). Checkmark icon (Unicode ✓ or CSS-only checkmark, 48px, `--cma-success` color). Heading "Results sent" in `--cma-success`. Body text. No spinner. No buttons. | Auto-dismiss after exactly 5 seconds via `setTimeout` → transitions to `SUBMIT_DONE` screen. User CAN screenshot the scoreboard (it's visible but dimmed). |
| `error` | Same overlay backdrop. Card with destructive indicator: ⚠ (Unicode warning, 48px, `--cma-destructive`). Heading "Unable to submit your results". Body text. Secondary message in `--cma-text-secondary`. "Retry Submission" button (styled as primary: `--cma-accent` background, 8px radius, 600 weight, 1rem font, `--cma-text-primary` color) centered below body. | "Retry Submission" button resets attempt counter to 0 and re-initiates `submitResults()`. User remains on scoreboard screen. |

#### 2. `SubmitDoneScreen` (new: `src/components/SubmitDoneScreen.jsx`)

Terminal screen. No navigation, no back button, no interactive elements.

| Visual | Specification |
|--------|---------------|
| Layout | Centered vertically + horizontally within viewport. Uses `.cma-screen` for max-width and padding. No scroll needed. |
| Heading | "Thank you, {name}" — 24px, weight 700, line-height 1.2, `--cma-text-primary`, no margin-bottom |
| Body | Single paragraph — 16px, weight 400, line-height 1.6, `--cma-text-secondary`, max-width 560px, centered text |
| Vertical rhythm | Heading → 24px gap → Body. Total block centered in viewport. |
| Background | `--cma-bg-deepest` (inherited from body) |
| Behavior | Clears `sessionStorage` identity on mount (`cma_identity_v1` key). Does NOT render if identity is null (edge case: screen only reachable from successful submission which guarantees identity exists). |

**Component signature:**
```
SubmitDoneScreen({ identity: { name } })
```
Imported in App.jsx and rendered in `SCREENS.SUBMIT_DONE` case.

## Screen State Flow Diagram

```
SCOREBOARD mounts
  ↓ (500ms delay for GSAP stagger)
submissionPhase = 'submitting' (overlay appears)
  ↓ POST attempt 1 → 5xx/error
submissionPhase stays 'submitting', attemptCount = 2 (overlay updates text)
  ↓ POST attempt 2 → 5xx/error
submissionPhase stays 'submitting', attemptCount = 3
  ↓ POST attempt 3 → 200 OK
submissionPhase = 'success' (checkmark + green card)
  ↓ 5-second setTimeout
submissionPhase = 'done' (internal cleanup)
  ↓ goToScreen(SCREENS.SUBMIT_DONE)
SUBMIT_DONE mounts → clears sessionStorage identity
  ↓ terminal (no further transitions)

  ═══ ERROR PATH (all 3 exhausted) ═══
  ↓ POST attempt 3 → error
submissionPhase = 'error' (⚠ card + retry button)
  ↓ user clicks "Retry Submission"
attemptCount reset to 1, submissionPhase = 'submitting'
  ↓ (retry cycle repeats)
```
