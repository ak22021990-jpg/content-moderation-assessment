    Create the file with the exact pattern from 06-RESEARCH.md (lines 242-253):
    ```javascript
    const BASE_URL = import.meta.env.VITE_VIDEO_BASE_URL || ''

    export function resolveVideoUrl(relativePath) {
      if (!BASE_URL || !relativePath) return relativePath || ''
      const base = BASE_URL.replace(/\/+$/, '')
      const path = relativePath.replace(/^\/+/, '')
      return `${base}/${path}`
    }
    ```
    This is a pure function with no side effects. The `import.meta.env.VITE_VIDEO_BASE_URL` is statically replaced by Vite at build time, so the empty-string fallback gets tree-shaken when the env var is undefined.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build-time .env → bundled JS | VITE_VIDEO_BASE_URL is statically replaced by Vite at build time. Runtime modification is not possible. |
| answerKeys.json → submission payload | answerKeyVersion ties each candidate result to its answer key version. Tampering with answer keys post-launch without version bump corrupts cross-cohort scoring. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03a | Tampering | VITE_VIDEO_BASE_URL in .env | mitigate | Build-time static replacement cannot be modified at runtime. .env is git-ignored on this project — production .env exists only in CI secrets. |
| T-06-03b | Tampering | answerKeys.json version | mitigate | Per-video answerKeyVersion + root version bump on every edit. Submission payload carries version string — cross-cohort comparison requires matching versions. |
| T-06-03c | Information Disclosure | answerKeys.json bundled in JS | accept | Known limitation per SCORE-08. Answer keys accepted as obscurity-not-security tradeoff for v1. v2 may move to server-side scoring. |
| T-06-03d | Repudiation | _status: placeholder bypass | mitigate | CI test `playlist.test.js` validates _status values. Production deploy blocked until all entries are "real". Documented in pipeline. |
| T-06-SC | Tampering | npm installs | mitigate | slopcheck audit in 06-RESEARCH.md — zero new packages. All changes are to existing source files only. |
</threat_model>
