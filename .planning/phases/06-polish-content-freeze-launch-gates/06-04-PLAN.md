    - **κ ≥ 0.6:** Category passes. No action needed.
    - **κ < 0.6:** Category fails. Root-cause analysis required:
      - Are L2 definitions ambiguous? → Tighten wording in taxonomy.json.
      - Is video content unclear about which violation is present? → Re-edit video or overlay to make violation more explicit.
      - Is answer key wrong? → Correct answer key, re-run kappa on just that video.
    - After remediation, re-run kappa for the failed category only.

    - `docs/kappa-calibration.md` must contain:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Video producer → repo | Video files must be audited for brand IP before commit. Binary content is opaque to text-based grep — manual review required. |
| Rater → answer key | 3 independent raters provide ground truth. Kappa < 0.6 indicates answer key or video ambiguity — must remediate. |
| Client reviewer → answer key | Client sign-off is the final authority on what constitutes a violation. Sign-off date is the version anchor. |
| R2 bucket → public internet | R2 bucket must be publicly readable for video serving. Bucket name and URL must not contain brand identifiers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04a | Information Disclosure | Video overlays/spoken text | mitigate | Manual frame-by-frame brand audit of all 5 videos per `docs/brand-guardrails.md` 55-string list. Chapter labels and VTT text included in grep scan. Documented in video-manifest.md per-video brand-audit section. |
| T-06-04b | Repudiation | Answer key version tracking | mitigate | Per-video answerKeyVersion + root version bump on every edit. Client sign-off date recorded as version anchor. Any post-sign-off change requires version bump documented in commit message. |
| T-06-04c | Elevation of Privilege | R2 bucket public access | mitigate | Bucket is read-only public. No write credentials in repo. Upload is manual via Cloudflare dashboard. Bucket name is generic (no brand reference). |
| T-06-04d | Information Disclosure | AI-gen video metadata | mitigate | AI-gen provider ToS verification date documented per video. If provider ToS prohibits commercial use, that provider is not used. Screenshots of ToS saved in docs/ for audit trail. |
| T-06-SC | Tampering | npm installs | mitigate | slopcheck audit in 06-RESEARCH.md — zero new packages. Content production uses external tools (ffmpeg, AI-gen platforms) outside npm ecosystem. |
</threat_model>
